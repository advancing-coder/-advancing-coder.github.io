PTA 旅游规划–Dijkstra算法应用

题目描述

有了一张自驾旅游路线图，你会知道城市间的高速公路长度、以及该公路要收取的过路费。现在需要你写一个程序，帮助前来咨询的游客找一条出发地和目的地之间的最短路径。如果有若干条路径都是最短的，那么需要输出最便宜的一条路径。

输入格式

输入说明：输入数据的第1行给出4个正整数N、M、S、D，其中N（2≤N≤500）是城市的个数，顺便假设城市的编号为0~(N−1)；M是高速公路的条数；S是出发地的城市编号；D是目的地的城市编号。随后的M行中，每行给出一条高速公路的信息，分别是：城市1、城市2、高速公路长度、收费额，中间用空格分开，数字均为整数且不超过500。输入保证解的存在

样例

4 5 0 3
0 1 1 20
1 3 2 30
0 3 4 10
0 2 2 20
2 3 1 20

输出格式

在一行里输出路径的长度和收费总额，数字间以空格分隔，输出结尾不能有多余空格。

样例

3 40

对于Dijkstra算法的总结

第一：它解决的是单源路径的最短距离，对于有闭合的环他是不适的

第二：实现的方式是通过一个dist数组实现，运用了贪心的思想，它表示的是距离源点s的最短距离，每次筛选出一个最短的距离，放入dist数组那么它最后到终点一定是最短的距离的

第三：在解决最短距离的同时可能有其他的权值例如花费的钱等待在改进距离时加上一个花费cost的数组

第四：解析一下代码 最外层的for循环是因为每做一次它的动作是向dist数组加入一个点，把这个最短距离的点作为下一次的起点，加入的具体动作就是将visited数组设为true，就是已经被访问 所以需要一个for循环加进去 第二个for循环是为了找距离起点距离最短的点，找他是为了更新它的后继点的距离，第三个for循环是以u为起点，遍历他的邻接点，更新下一次的最短的距离。
第五：最后dist数组存的就是以s为起点到各个点距离的最小值了，需要输出谁就直接用其索引值访问即可。

力扣 填充每个节点的下一个右侧节点指针

题目描述

给定一个完美二叉树，其所有叶子节点都在同一层，每个父节点都有两个子节点。二叉树定义如下：

struct Node {
int val;
Node *left;
Node *right;
Node *next;
}
填充它的每个 next 指针，让这个指针指向其下一个右侧节点。如果找不到下一个右侧节点，则将 next 指针设置为 NULL。
初始状态下，所有 next 指针都被设置为 NULL。

AC 代码

总结：解题的关键在于递归，而如何拆分递归的链条就十分关键了：那么归结起来无非三种情况，拆分一个小的节点来看的话，第一种当然是最特殊的root为NULL，其他的要么就是链接左边，要么就是链接右边，右边的话好说，就是判断它在不在，而要链左边的话，肯定是这边的右子树，链接隔壁root的左子树，那么就有可能为空 要么就去找隔壁的左子树，用next可以方便的寻找隔壁然后就把他连接上

力扣445两数相加

题目描述

给你两个 非空 链表来代表两个非负整数。数字最高位位于链表开始位置。它们的每个节点只存储一位数字。将这两数相加会返回一个新的链表。

你可以假设除了数字 0 之外，这两个数字都不会以零开头。

实例

输入：
(7 -> 2 -> 4 -> 3) + (5 -> 6 ->4)
输出：7 -> 8 -> 0 -> 7

解题思路

把两个链表反转然后做加法,再把和反转一下就得到结果

主要是在做加法的时候对于进位的一些细节处理，以及链表没有头节点的处理问题，个人感觉在自己做的时候还是把链表设置一个头节点比较舒服，因为题目给出了没有头节点所以在处理的时候有一些细节的体现

还有就是在链表反转函数的时候注意一下return的细节

AC代码

代码解析

在链表相加函数中,是把结果串在l1上的，所以先记录他的头节点
当l1和l2都非空的时候我们把两个相加只需更改l1的val值
进位需要考虑l1和l2是否为空的情况是否需要malloc一个新的节点这些细节都是非常重要的。

踩过的一个坑就是c++ malloc一段内存必须要释放出来，不然要报错，因为没有仔细看报错的具体信息，一直以为是用了空指针，造成了很多麻烦，还有就是涉及指针的调试都不太会，不会查看内存窗口。
