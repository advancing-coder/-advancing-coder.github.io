发表于 2020-03-20

算法—力扣组数总和

题目描述

给定一个无重复元素的数组 candidates 和一个目标数 target ，找出 candidates 中所有可以使数字和为 target 的组合。
candidates 中的数字可以无限制重复被选取。
说明：所有数字（包括 target）都是正整数。解集不能包含重复的组合。

示例 1:

输入: candidates = [2,3,6,7], target = 7,
所求解集为:
[
[7],
[2,2,3]
]

示例 2:

输入: candidates = [2,3,5], target = 8,
所求解集为:
[
[2,2,2,2],
[2,3,3],
[3,5]
]

题目分析

这道题和之前的二进制枚举有所区别,因为数组中的元素是可以重复选取的
所以用二进制枚举的方法显然是不可行的,我想的这个好像有点像动态规划的那种思想,就是我首先遍历数组中的元素,先取一个如果这个值比target小那么就把他拿进来,计入一个数组。因为元素是可以重复的,有可能会再次选这个元素,也有可能往下遍历。如果下一个值加上之前的数的和sum+=num[i]<target那么就继续加进去,如果遍历完整个数组还是找不到匹配的值，那么就需要将前一个push_back()继续寻找新的值。

这个就好像有一种递归的感觉在里面。

过程分析

下面继续分析一下这个递归调用的过程,看看pop_back()的具体实现过程

我以[3,2,4] target=7进行分析

step1:
temp=[3] target=4 (后面还有一个pop_back()等待递归调用完成之后执行)

step2:
现在是第一次递归
temp=[3,3] target=1 (后面还有一个pop_back()等待递归调用完成之后执行)

step3:
第二次递归
遍历没有找到元素,执行pop_back()
现在temp=[3] target=4 返回上一级

step4
step2下的for循环
temp=[3,2] target=5 (后面还有一个pop_back()等待递归调用完成之后执行)
在这一次接下来的递归中还是没发现最终会pop_back()返回到step2的for循环

step5
step2下的for循环
temp=[3,4] target=7, 找到了就res.push_back()
接下来他还会继续遍历随着遍历找不到匹配的东西,之前的匹配的temp中的元素会一步一步的被pop_back()所以不会担心temp没有被清空的问题

总结

这个应该还有值得改进的地方,如果先给数组排一个序如果找到的sum>target那么就可以提前结束这个查找了。
